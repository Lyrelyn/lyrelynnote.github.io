## 分支策略

推荐使用 Git Flow 或 GitHub Flow，根据团队规模和发布节奏选择。

## 提交信息规范

采用 Conventional Commits 格式：`type(scope): description`，有助于自动生成 changelog。

## 常见场景

解决合并冲突、Cherry-pick、交互式 rebase 等实用技巧。
