## Why Next.js for a blog?

Next.js has become the go-to framework for React applications, and it's an excellent choice for building a blog. Here's why:

- **Static Site Generation (SSG)** — Pre-render all your posts at build time for blazing fast page loads
- **Incremental Static Regeneration (ISR)** — Update content without a full rebuild
- **App Router** — React Server Components by default, reducing client-side JavaScript
- **Built-in image optimization** — Automatic WebP/AVIF conversion and responsive images
- **SEO primitives** — Built-in metadata API, sitemap generation, and more

## Content management with MDX

MDX lets you write Markdown and embed React components in the same file. This means you can create rich, interactive content without leaving your content files.

```tsx
// Example: a custom Callout component used directly in MDX
export function Callout({ type, children }) {
  const icons = {
    info: 'ℹ️',
    warning: '⚠️',
    success: '✅',
  };
  return (
    <div className={`callout callout-${type}`}>
      <span>{icons[type]}</span>
      {children}
    </div>
  );
}
```

## Performance is key

A few things that make this setup fast:

### 1. Server Components

Most of the page content never ships JavaScript to the client. Only interactive islands like the theme toggle need client-side hydration.

### 2. Image optimization

All images are automatically optimized to WebP format with responsive sizes. Lazy loading ensures below-the-fold images don't block the initial render.

### 3. Minimal CSS

Tailwind CSS purges unused styles at build time. The production CSS bundle is typically under 10KB.
